# Метрики, отчётность и полнота плана (как в крупном проекте)

> [← Главный план](main.md)

В этом разделе: **какие метрики считать**, **когда и с какой периодичностью**, **что нужно для их расчёта** (источники данных), **целевые значения**, ритм отчётности. Плюс блок в стиле «большого плана»: ключевые KPI, unit economics, риски/предположения, чек-лист перед запуском и сводка бюджета по фазам.

## 1. Основные маркетинговые и бизнес-метрики

### 1.1 Сводная таблица: что считать, когда, откуда данные, цели


| Метрика                                 | Что считать                                                         | Когда считать                        | Что нужно для расчёта                                                                                                                   | Цель / ориентир                                                                                                                                                       |
| --------------------------------------- | ------------------------------------------------------------------- | ------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Лиды (кол-во)**                       | Заявки + звонки по источникам                                       | Ежедневно / еженедельно              | [CRM](03-crm.md) с полем «Источник»; [коллтрекинг](07-analytics-and-tools.md) → CRM                                       | План по лидам в месяц (из [бюджета и CPL](06-marketing-channels.md))                                                                      |
| **CPL (стоимость лида)**                | Расход на канал (€) / кол-во лидов                                  | Еженедельно / ежемесячно             | Расходы по каналам (рекламные кабинеты), лиды по источникам из CRM                                                                      | CPL ≤ CPL_макс (из [зависимостей](08-strategy-discounts-dependencies.md))                                             |
| **Конверсия лид → заказ**               | (Заказы по источнику / Лиды по источнику) × 100%                    | Еженедельно / ежемесячно             | CRM: лиды и сделки со статусом «Выполнено», с указанием источника                                                                       | 20–30% и выше; рост при улучшении оффера и обработки                                                                                                                  |
| **CAC (стоимость привлечения клиента)** | Расход на канал (€) / кол-во новых клиентов по каналу               | Ежемесячно                           | То же + количество закрытых сделок по источникам                                                                                        | CAC ≤ маржа на заказ × доля на привлечение (см. [нормативы](08-strategy-discounts-dependencies.md))                               |
| **Средний чек (AOV)**                   | Выручка за период / кол-во заказов                                  | Еженедельно / ежемесячно             | [CRM](03-crm.md) или учёт: сумма сделок, количество заказов                                                                             | Стабильность или рост; связь с [прайсом и пакетами](02-services-pricing.md)                                                                                           |
| **Выручка**                             | Сумма оплаченных заказов (€)                                        | Ежедневно / еженедельно / ежемесячно | CRM или бухгалтерия                                                                                                                     | План по выручке; [точка безубыточности](02-services-pricing.md)                                                                       |
| **Маржа (валовая)**                     | Выручка − себестоимость заказов (€ и %)                             | Ежемесячно                           | Выручка; себестоимость из [ERP](04-erp-inventory.md) или расчёт по нормам из [карточек услуг](02-services-pricing.md) | Целевая доля маржи (например 35–50%); не падать при скидках ниже [минимальной наценки](08-strategy-discounts-dependencies.md) |
| **ROI рекламы**                         | (Выручка по каналу − Расход на канал) / Расход на канал × 100%      | Ежемесячно                           | Привязка выручки к источнику (сделки в CRM с источником); расходы по каналам                                                            | > 0 для безубыточности; > 50–100% для здорового канала                                                                                                                |
| **LTV (lifetime value)**                | Средняя выручка с одного клиента за всё время (или за 12 мес)       | Ежеквартально                        | CRM: история заказов по клиентам                                                                                                        | Рост при [retention](09-marketing-seo-extensions.md); LTV > CAC в 3+ раза — хороший ориентир                               |
| **Доля повторных клиентов**             | Клиенты с 2+ заказами / все клиенты × 100%                          | Ежемесячно / ежеквартально           | CRM: подсчёт уникальных клиентов и повторных обращений                                                                                  | Рост при работе над удержанием                                                                                                                                        |
| **Конверсия по воронке**                | % перехода между этапами (лид → квалификация → предложение → заказ) | Ежемесячно                           | CRM: сделки по этапам и датам перехода                                                                                                  | Выявление «узких» этапов; улучшение скриптов и процессов                                                                                                              |
| **Трафик и цели на сайте**              | Визиты, просмотры, отправки формы, клики по телефону                | Еженедельно                          | [Аналитика](07-analytics-and-tools.md): маркетинговая (GA4, Matomo) — цели настроены; при необходимости продуктовая (PostHog) — воронки, записи сессий для разбора конверсии                                                                    | Рост трафика; конверсия визит → заявка/звонок                                                                                                                         |
| **Отзывы и рейтинг**                    | Кол-во отзывов, средний рейтинг (Google и др.)                      | Ежемесячно                           | Мониторинг вручную или сервисы; [репутация](09-marketing-seo-extensions.md)                            | Рост количества и поддержание рейтинга ≥ 4,0–4,5                                                                                                                      |

### 1.2 Что должно быть настроено, чтобы метрики считались


| Метрика                                 | Необходимые настройки                                                                                                                                                 |
| --------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Лиды по источникам, CPL, конверсия, CAC | В [CRM](03-crm.md) у каждого лида и сделки — поле **Источник**; лиды с сайта и из [коллтрекинга](07-analytics-and-tools.md) попадают в CRM с источником |
| Выручка, средний чек                    | В CRM или учёте — сумма по сделке при закрытии; при необходимости экспорт в отчёт                                                                                     |
| Маржа                                   | [ERP](04-erp-inventory.md) или расчёт: списание материалов по заказам + труд по нормам; либо фиксированная средняя доля маржи для оценки                              |
| ROI по каналу                           | Возможность привязать выручку к источнику (сделки с заполненным источником); учёт расходов по каналам (рекламные кабинеты, коллтрекинг)                               |
| LTV, повторные клиенты                  | В CRM — одна карточка клиента и все его сделки с датами                                                                                                               |
| Конверсия по воронке                    | В CRM — этапы сделки и даты перехода (или дата создания/закрытия по этапам)                                                                                           |
| Трафик и цели                           | В [аналитике](07-analytics-and-tools.md) — цели на отправку формы и клик по телефону (GA4/Matomo); UTM или источник передаётся с сайта; при использовании PostHog — воронки и события для детального разбора конверсии                                             |


Без корректного **источника** в CRM маркетинговые метрики (CPL, CAC, конверсия по каналу, ROI) будут неверными или неполными. Ответственный за сбор и отчётность по метрикам целесообразно закрепить в [должностных инструкциях](13-job-descriptions.md) (менеджер, владелец).

## 2. Unit economics (сводка в одном месте)

Как в крупных планах: ключевые показатели экономики единицы (один клиент, один заказ).


| Показатель                               | Формула                                                            | Откуда данные                                                                   | Норма / цель                                                                                                                                  |
| ---------------------------------------- | ------------------------------------------------------------------ | ------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| **CAC**                                  | Расходы на привлечение / кол-во новых клиентов                     | Реклама + CRM по источникам                                                     | CAC ≤ 20–30% от LTV или ≤ доля маржи на заказ ([норматив](08-strategy-discounts-dependencies.md))         |
| **LTV**                                  | Средняя выручка с клиента за период (или сумма заказов по клиенту) | CRM: история по клиентам                                                        | LTV / CAC ≥ 3 — устойчивая модель; рост за счёт [удержания](09-marketing-seo-extensions.md)        |
| **Payback (окупаемость CAC)**            | CAC / (Маржа с клиента в месяц или за заказ)                       | CAC из отчётов; маржа из [прайса](02-services-pricing.md) и фактических заказов | Окупаемость за 3–12 месяцев в зависимости от сегмента                                                                                         |
| **Вклад на заказ (contribution margin)** | Цена − переменные затраты (материалы + прямая доля труда) на заказ | [Карточки услуг](02-services-pricing.md) или ERP              | Положительный и достаточный для покрытия постоянных расходов ([точка безубыточности](02-services-pricing.md)) |
| **Доля маркетинга в выручке**            | Расходы на маркетинг (€) / Выручка (€) × 100%                      | Бюджет рекламы и выручка за период                                              | В рамках [норматива](08-strategy-discounts-dependencies.md) (например 5–15%)                              |


Эти показатели пересматривают при изменении цен, скидок, каналов и объёма рекламы.

## 3. Ритм отчётности и кому что нужно


| Периодичность     | Содержание отчёта                                                                                                                                                                                                    | Кто использует                                                                                                                            |
| ----------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| **Еженедельно**   | Лиды по источникам, заявки, звонки; конверсия лид → заказ за неделю; трафик и цели на сайте (если настроены)                                                                                                         | Ответственный за маркетинг; корректировка рекламы и обработки лидов                                                                       |
| **Ежемесячно**    | CPL, CAC, средний чек, выручка, маржа по каналам; ROI рекламы; конверсия по этапам воронки; сравнение с планом и с прошлым месяцем                                                                                   | Принятие решений по бюджету, приоритетам каналов, [нормативам](08-strategy-discounts-dependencies.md) |
| **Ежеквартально** | LTV, доля повторных клиентов; unit economics (CAC, LTV, payback); пересмотр [стратегии и нормативов](08-strategy-discounts-dependencies.md); [аудит](09-marketing-seo-extensions.md) | Стратегические решения; план на следующий квартал                                                                                         |


Дашборд (таблица в Google Sheets или отчёты из CRM/аналитики) лучше собрать один раз под эти метрики и обновлять по расписанию. Рекомендации по составу дашборда и по другим инструментам оптимизации (внутренний аудит процессов, SLA, рекламации) — в [Оптимизация бизнес-процессов](14-business-process-optimization.md).

## 4. Ключевой KPI и North Star (опционально)

В «больших» планах часто задают один главный показатель успеха:

- **Вариант для детейлинга:** «Выручка за месяц» или «Количество оплаченных заказов в месяц» — простые и измеримые.
- **Альтернатива:** «Маржинальная прибыль за месяц» — учитывает и объём, и наценку.

Остальные метрики (лиды, CPL, CAC, конверсия, средний чек) — **драйверы** этого KPI: через них вы понимаете, что двигать для роста.

## 5. Риски и предположения (что может пойти не так)

Краткий список того, от чего зависит план и что стоит отслеживать:


| Предположение / риск                                       | Что делать                                                                                                                                     |
| ---------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| CPL и конверсия будут в ожидаемых границах                 | Тестовые кампании с ограниченным бюджетом; при стабильно высоком CPL — оптимизация креативов, посадочных, таргетинга                           |
| Клиенты оставляют контакт и источник в CRM (в т.ч. звонки) | Обязательная настройка [коллтрекинга](07-analytics-and-tools.md) и обучение менеджеров проставлять источник при ручном вводе     |
| Маржа не падает из-за неконтролируемых скидок              | [Правила скидок](08-strategy-discounts-dependencies.md) и лимиты; контроль среднего чека и маржи по отчётам   |
| Постоянные расходы и загрузка соответствуют плану          | Мониторинг выручки и [точки безубыточности](02-services-pricing.md); при просадке — пересмотр расходов или цен |
| Сайт и формы работают, лиды доходят до CRM                 | Регулярная проверка форм и интеграций; цели в аналитике на отправку формы                                                                      |


При значительном отклонении факта от плана (например CPL в 2 раза выше расчётного) — пересмотр бюджета, каналов или оффера.

## 6. Сценарии корректировки (план Б)

Если факт сильно расходится с планом, действовать по сценариям:

| Ситуация | Что проверить и что делать |
|----------|-----------------------------|
| **CPL стабильно выше CPL_макс** | Снизить бюджет по каналу или перераспределить; сфокусироваться на 1 канале с лучшим CPL; улучшить посадочную (оффер, форма, скорость); проверить, что все лиды (в т.ч. звонки) попадают в CRM с источником — иначе CPL завышен. |
| **Конверсия лид → заказ низкая** | Разобрать причины отказов (цена, сроки, не тот объём); усилить скрипты и КП ([УТП](08-strategy-discounts-dependencies.md)); A/B по форме и посадочной; проверить скорость обработки лидов. |
| **Мало лидов при заданном бюджете** | Проверить показы и клики (доходят ли объявления); пересмотреть таргетинг и ключевые слова; протестировать другие креативы и заголовки; при необходимости временно повысить бюджет на тест. |
| **Выручка ниже точки безубыточности** | Пересмотреть [постоянные расходы](02-services-pricing.md) и загрузку; варианты: снижение расходов, акции для заполнения слотов (в рамках [правил скидок](08-strategy-discounts-dependencies.md)), рост цен там, где допустимо. |

Использовать [критерии успеха по фазам](main.md) из главного плана: при провале фазы — сначала план Б, затем при необходимости пересмотр целей и нормативов.

## 7. Чек-лист перед запуском (Go-to-Market)

Перед тем как включать платную рекламу и считать маркетинговые метрики по плану, стоит проверить:


| №   | Проверка                                                                                              | Ссылка в плане                                                                                                                                                  |
| --- | ----------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | Прайс и себестоимость утверждены; известна маржа и CAC_макс                                           | [Услуги и прайс](02-services-pricing.md), [Зависимости](08-strategy-discounts-dependencies.md)                  |
| 2   | Правила скидок зафиксированы; минимальная цена не нарушается                                          | [Скидки](08-strategy-discounts-dependencies.md)                                                                                |
| 3   | В CRM есть поля «Источник», этапы воронки; лиды с сайта и из коллтрекинга попадают в CRM с источником | [CRM](03-crm.md), [Аналитика и коллтрекинг](07-analytics-and-tools.md)                                                                                          |
| 4   | На сайте: актуальные цены, УТП, форма заявки; UTM или способ передачи источника на посадочных         | [Сайт](05-website.md), [Стратегия](08-strategy-discounts-dependencies.md)                                                              |
| 5   | В аналитике настроены цели: отправка формы, клик по телефону                                          | [Аналитика](07-analytics-and-tools.md)                                                                                                                          |
| 6   | Коллтрекинг настроен: подмена номера, передача источника в CRM при звонке                             | [Коллтрекинг](07-analytics-and-tools.md)                                                                                                          |
| 7   | Определён ответственный за маркетинг и ритм отчётов (неделя/месяц)                                    | [Расширения плана](09-marketing-seo-extensions.md)                                                                                         |
| 8   | Бюджет на рекламу на первый месяц определён; не превышает плановый норматив                           | [Нормативы](08-strategy-discounts-dependencies.md), [Расчёт кампаний](06-marketing-channels.md) |


После прохождения чек-листа можно запускать кампании и считать CPL/CAC и остальные метрики по плану.

## 8. Бюджет по фазам (сводка для плана)

Чтобы план был полным, как в крупном проекте, полезно зафиксировать не только «сколько на маркетинг в месяц», но и **как бюджет распределён по фазам** (разовые и повторяющиеся расходы).


| Фаза            | Статьи расходов (пример)                                           | Тип                            | Примечание                                                                                      |
| --------------- | ------------------------------------------------------------------ | ------------------------------ | ----------------------------------------------------------------------------------------------- |
| **Основа**      | Юрлицо, консультации, возможно бренд/логотип                       | Разовые                        |                                                                                                 |
| **Основа**      | Карточки услуг, прайс (время/подрядчик)                            | Разовые                        |                                                                                                 |
| **Системы**     | CRM, аналитика, коллтрекинг (подписки)                             | Ежемесячно                     | Уже в [нормативах](08-strategy-discounts-dependencies.md)   |
| **Системы**     | Сайт: разработка/шаблон, контент, хостинг                          | Разовые + ежемесячно (хостинг) |                                                                                                 |
| **Привлечение** | Реклама (Google, Meta): тест затем масштаб                         | Ежемесячно                     | По [расчёту от CPL и лидов](06-marketing-channels.md)               |
| **Привлечение** | Коллтрекинг, дополнительные инструменты                            | Ежемесячно                     |                                                                                                 |
| **Масштаб**     | Рост рекламного бюджета; контент, ретаргетинг, возможно подрядчики | Ежемесячно                     | В рамках [норматива](08-strategy-discounts-dependencies.md) |
| **Регулярно**   | Аудит, доработки сайта, обучение                                   | Периодически                   | [Рекомендации](09-marketing-seo-extensions.md)                  |


Конкретные суммы подставляются из ваших [постоянных расходов](02-services-pricing.md) и [нормативов](08-strategy-discounts-dependencies.md); эта таблица задаёт структуру, чтобы ничего не упустить.

## 9. Связь с остальными разделами плана

- [Услуги и прайс](02-services-pricing.md) — себестоимость, маржа, точка безубыточности; нужны для целей по марже и для unit economics.
- [CRM](03-crm.md) — источник лида, этапы воронки, история по клиентам; без этого не посчитать CPL, CAC, конверсию, LTV.
- [Аналитика и коллтрекинг](07-analytics-and-tools.md) — учёт звонков и целей; иначе часть лидов и конверсий теряется.
- [Стратегия и зависимости](08-strategy-discounts-dependencies.md) — нормативы, CAC_макс, CPL_макс; цели для метрик и границы бюджета.
- [Каналы и кампании](06-marketing-channels.md) — расчёт бюджета и целевых лидов; фактические CPL/CAC сравниваются с этими целями.
- [Расширения плана](09-marketing-seo-extensions.md) — ритм отчётов, ответственный, аудит; дополняют раздел «Процессы и команда».
- [Оптимизация бизнес-процессов](14-business-process-optimization.md) — дашборд, внутренний аудит процессов, SLA, NPS/качество; связь метрик с операционными регламентами.
- [Интеграция бизнес-процессов](15-business-process-integration.md) — CRM–ERP–бухгалтерия, выручка и маржа по данным из CRM/ERP/бухгалтерии; программа лояльности и метрики LTV, доля повторных.
